## Key Decisions Made

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| Cloud Run for 9router API | Fastest (0.38s TTFB), 1 vCPU, pay-per-use (~free) | VM e2-micro (too slow), VPS (adds hop) |
| Direct Cloud Run URL for AI tools | No latency overhead for API calls | Custom domain adds +0.55s per request |
| Custom domain only for dashboard | Dashboard access is infrequent, latency OK | Direct URL only |
| Cloudflare Tunnel for custom domain | SSL + custom domain without domain mapping issues | Cloudflare proxy (525 error), domain mapping (404), NPM |

## Important Notes

- **Cloud Run cold start**: ~2-5 seconds when container scales from 0. First request after idle will be slow.
- **Data persistence**: Cloud Run is stateless. 9router stores data in memory/local — data resets on container restart. Consider adding persistent storage (Cloud Storage, Firestore) if needed.
- **The Dockerfile** at `/tmp/9router-deploy/` is the working version. Keep it for redeployment.
- **Tunnel route** `combo.maxai.site` currently points to Cloud Run URL via VPS connector. Settings: Type=HTTP, URL=`combo-cliproxy-1097063009049.us-west1.run.app`, HTTP Host Header override applied.
